## Step 2 — Set up Tailwind (CLI version — no bundler needed)

```bash
npm init -y
npm install -D tailwindcss
npx tailwindcss init
```

This creates `package.json` and `tailwind.config.js`. Open `tailwind.config.js` and set it up like this:

```js
module.exports = {
  content: ["./*.html", "./products/*.html", "./js/*.js"],
  theme: {
    extend: {
      colors: {
        gold: '#C9A44C',
        charcoal: '#0D0D0D',
      },
      fontFamily: {
        serif: ['Playfair Display', 'serif'],
        sans: ['Poppins', 'sans-serif'],
      },
    },
  },
  plugins: [],
}
```

Create a folder `src/` with a file `src/input.css`:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

In `package.json`, add these two scripts:

```json
"scripts": {
  "dev": "tailwindcss -i ./src/input.css -o ./dist/output.css --watch",
  "build": "tailwindcss -i ./src/input.css -o ./dist/output.css --minify"
}
```

Run `npm run dev` while you work locally — it recompiles CSS automatically as you edit. In every HTML file's `<head>`, link the compiled file:

```html
<link rel="stylesheet" href="/dist/output.css">
```

## Step 4 — Set up Git

Create a `.gitignore` file in the root:

```
node_modules/
dist/
.DS_Store
```

(`dist/` is excluded because Netlify will generate it automatically during deployment — no need to commit compiled CSS.)

Then:

```bash
git init
git add .
git commit -m "Initial commit: ShelemJ website scaffold"
```

## Step 5 — Push to GitHub

**Easiest way — inside VS Code:**
Click the Source Control icon in the left sidebar → **Publish to GitHub** → choose public or private → name it `shelemj-website`. VS Code creates the repo and pushes for you.

**Or via terminal:**
1. Go to github.com/new → create a repo called `shelemj-website` → do **not** initialize with a README
2. Then:
```bash
git remote add origin https://github.com/Justixxprime/shelemj-website.git
git branch -M main
git push -u origin main
```

---

## Step 6 — Connect the repo to Netlify

1. Go to **app.netlify.com** → sign in with GitHub
2. **Add new site** → **Import an existing project** → **Deploy with GitHub**
3. Authorize Netlify, select `shelemj-website`
4. Build settings:
   - **Build command:** `npm run build`
   - **Publish directory:** `.`
5. Click **Deploy site**

Netlify installs dependencies, builds your CSS, and gives you a live link like `random-name-123.netlify.app` within about a minute.

**Rename it** (optional, free): Site settings → General → Change site name → e.g. `shelemj` → link becomes `shelemj.netlify.app`. Good to send your aunt before the real domain is live.

## Step 8 — Ongoing workflow from here

Every time you make changes locally:

```bash
git add .
git commit -m "describe what changed"
git push
```

Netlify auto-deploys within about a minute — no manual redeploy needed.

**To show your aunt a draft without touching the live site:**
```bash
git checkout -b draft-v2
git push origin draft-v2
```
Netlify auto-generates a separate preview URL for that branch. Send her that link. Once approved, merge it into `main`.

## Step 10 — Point the domain to Netlify

**Recommended method — Netlify DNS (simplest):**

1. In Netlify: Site settings → **Domain management** → **Add a domain** → enter `shelemjresourcesltd.com`
2. Netlify shows you 4 nameservers (something like `dns1.p05.nsone.net`, etc. — exact values shown on your dashboard)
3. Go to Namecheap → Domain List → **Manage** next to the domain → **Nameservers** section → select **Custom DNS** → paste in Netlify's 4 nameservers → save
4. Wait for propagation (usually well under 24 hours)
5. Once verified, Netlify automatically issues a free HTTPS certificate — you'll see the padlock

*Alternative:* if you ever want a separate email provider with its own DNS records, you can instead keep Namecheap's DNS and just add the A record + CNAME Netlify provides, rather than switching nameservers entirely. Only needed if Step 10's simple method causes conflicts with something else you're running on the domain.
